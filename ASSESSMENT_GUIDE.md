This guide provides additional context and tips for completing the assessment.

## Task 1: Backend API - Detailed Requirements

### MongoDB Schema
Your Task model should look something like this:

```javascript
{
  title: { type: String, required: true },
  description: { type: String },
  status: { 
    type: String, 
    enum: ['pending', 'in-progress', 'completed'],
    required: true,
    default: 'pending'
  },
  priority: {
    type: String,
    enum: ['low', 'medium', 'high'],
    required: true,
    default: 'medium'
  },
  dueDate: { type: Date },
  // createdAt and updatedAt will be auto-generated by timestamps: true
}
```

### API Endpoints Examples

**GET /api/tasks**
- Query params: `?status=pending&priority=high`
- Response: Array of tasks

**GET /api/tasks/:id**
- Response: Single task object
- Error: 404 if not found

**POST /api/tasks**
- Body: `{ title, description?, status, priority, dueDate? }`
- Response: Created task object
- Error: 400 for validation errors

**PUT /api/tasks/:id**
- Body: `{ title?, description?, status?, priority?, dueDate? }`
- Response: Updated task object
- Error: 404 if not found, 400 for validation errors

**DELETE /api/tasks/:id**
- Response: Success message or deleted task
- Error: 404 if not found

### Error Handling
- Use try-catch blocks
- Return appropriate HTTP status codes (200, 201, 400, 404, 500)
- Provide meaningful error messages

## Task 2: Frontend - Detailed Requirements

### Component Structure
```
App.tsx (main component)
├── TaskList.tsx (displays all tasks)
│   └── TaskCard.tsx (individual task display)
└── TaskForm.tsx (create/edit form)
```

### State Management
- Use React hooks (useState, useEffect)
- Consider using Context API if needed (optional)
- Manage loading and error states

### API Integration
- Create a service file (`taskService.ts`) for all API calls
- Use axios or fetch
- Handle errors gracefully
- Show loading indicators

### Filtering
- Add filter dropdowns/buttons for status and priority
- Update the API call when filters change
- Show active filters

## Testing Your Application

### Backend Testing
1. Use Postman or Thunder Client (VS Code extension)
2. Test all endpoints
3. Test error cases (invalid IDs, missing fields, etc.)
4. Test filtering with query parameters

### Frontend Testing
1. Test all CRUD operations
2. Test filtering functionality
3. Test on different screen sizes
4. Test error scenarios (API down, invalid data, etc.)

## Common Pitfalls to Avoid

1. **Forgetting to connect routes in server.js**
   - Make sure to import and use your task routes

2. **Not handling async/await properly**
   - Use try-catch with async functions
   - Handle promise rejections

3. **Missing TypeScript types**
   - Define interfaces for all data structures
   - Type your function parameters and return values

4. **Not validating input**
   - Validate on both frontend and backend
   - Show user-friendly error messages

5. **Forgetting CORS**
   - Backend should allow requests from frontend origin
