# Assessment Repository - Hiring Team Guide

## Overview

This repository contains a MERN stack assessment designed to evaluate candidates' skills in:
- **M**ongoDB - Database design and queries
- **E**xpress - RESTful API development
- **R**eact - Frontend application with TypeScript
- **N**ode.js - Server-side JavaScript

## Assessment Structure

### Tasks (3-4 hours total)

1. **Backend API Development** (1.5-2 hours)
   - Build RESTful API with Express
   - MongoDB schema design
   - CRUD operations
   - Error handling and validation

2. **Frontend Application** (1.5-2 hours)
   - React with TypeScript
   - Vite build tool
   - API integration
   - Filtering and UI components

3. **Additional Feature** (30-45 minutes)
   - Choice of: Search, Statistics, or Sorting
   - Demonstrates problem-solving skills

## How to Use This Assessment

### 1. Share with Candidates

**Option A: GitHub Repository**
```bash
# Create a new repository from this template
# Share the repository URL with candidates
# Ask them to fork/clone and complete the assessment
```

**Option B: Zip File**
```bash
# Create a zip of this repository
# Remove this file (FOR_HIRING_TEAM.md) before sharing
# Share the zip file with candidates
```

### 2. Candidate Submission

Candidates should:
- Complete all tasks within 3-4 hours
- Commit their work with meaningful messages
- Share repository link or zip file
- Include a brief summary of their work

### 3. Evaluation Process

1. **Initial Review** (15 minutes)
   - Check if application runs
   - Verify all required features are present
   - Review code structure

2. **Detailed Review** (30-45 minutes)
   - Test all endpoints
   - Review code quality
   - Check TypeScript usage
   - Evaluate best practices

3. **Scoring** (15 minutes)
   - Use `EVALUATION_CRITERIA.md` rubric
   - Document strengths and weaknesses
   - Make recommendation

## What to Look For

### Green Flags ✅
- Clean, well-organized code
- Proper TypeScript usage
- Good error handling
- RESTful API design
- Responsive UI
- Meaningful commit messages
- Goes beyond requirements

### Red Flags ⚠️
- Code doesn't run
- Missing core functionality
- Poor code quality
- No error handling
- Security issues
- Plagiarism

### Yellow Flags 🟡
- Incomplete features (but understandable given time)
- Minor bugs (but good approach)
- Basic styling (but functional)

## Time Considerations

**Remember:** Candidates have 3-4 hours. Consider:
- Did they prioritize correctly?
- What did they choose to implement vs. skip?
- Is the code quality good even if incomplete?
- Do they show good problem-solving skills?
